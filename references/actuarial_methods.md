## 一、精算定价方法

### 1.1 四种主流定价方法对比

| 方法 | 适用场景 | 优点 | 局限性 |
|------|---------|------|--------|
| **纯保费法** | 有历史频率和强度数据 | 逻辑清晰，可溯源 | 需要分离频率/强度数据 |
| **损失率法** | 已有在售类似产品 | 操作简单，基于现有费率调整 | 依赖现有费率合理性 |
| **燃烧成本法** | 低频高损险种（灾险/责任险） | 适合巨灾风险、稀少历史数据 | 数据年限不足时误差大 |
| **类比法** | 全新险种、无历史数据 | 可快速出价 | 主观性强，需说明类比依据 |

### 1.2 纯保费法公式

```
纯保费（PP）= 索赔频率 × 平均赔款额
毛保费（GP）= PP ÷ (1 - 费用率 - 利润率 - 风险附加率)
```

**计算示例**：
- 历史索赔频率 = 0.05；平均赔款 = 10,000元；费用率20%；利润率10%；风险附加5%
- 纯保费 = 0.05 × 10,000 = 500元
- 毛保费 = 500 ÷ (1 - 0.35) = **769.23元**

### 1.3 损失率法公式

```
费率调整因子 = 目标赔付率 ÷ 实际赔付率
指示费率 = 当前费率 × 费率调整因子
目标赔付率 = 1 - 费用率 - 利润率
```

**计算示例**：
- 当前费率1000元；实际赔付率60%；目标赔付率65%
- 费率调整因子 = 65%/60% = 1.0833
- 指示费率 = **1,083.30元**

### 1.4 燃烧成本法步骤

```
Step 1: 确定观察期（通常5-10年）
Step 2: 计算历史平均年损失 = 总损失 ÷ 观察年数
Step 3: 趋势调整 = 平均年损失 × (1 + 趋势因子)^n
Step 4: 纯保费 = 趋势调整后损失 × (1 + 风险附加率)
```

## 二、分类费率厘定

### 2.1 单因素分析法

```
相对费率 = 某组纯保费 ÷ 基准组纯保费
调整后费率 = 基准费率 × 相对费率
```

### 2.2 广义线性模型（GLM）

| 模型类型 | 响应变量 | 分布 | 连接函数 | 应用 |
|---------|---------|------|---------|------|
| 频率模型 | 索赔次数 | 泊松分布 | log() | 预测索赔频率 |
| 强度模型 | 赔款额 | 伽马分布 | log() | 预测平均赔款 |
| 纯保费模型 | 总损失 | Tweedie | log() | 直接预测纯保费 |

**GLM模型结构**：`g(μ) = Xβ`（g为连接函数，X为风险因子矩阵，β为回归系数）

**模型选择**：AIC/BIC准则 + 偏差分析 + 残差诊断 + 交叉验证

### 2.3 信度理论（Bühlmann信度）

```
信度估计 = Z × 经验数据 + (1-Z) × 先验估计
Z = n / (n + k)   （n=观察期数；k=信度系数）
```

**应用**：新险种数据不足时，用行业数据作为先验估计，结合公司自身经验数据加权

## 三、准备金评估方法

### 3.1 准备金类型

| 类型 | 定义 | 评估方法 |
|------|------|---------|
| 未到期责任准备金 | 尚未到期保险责任对应的准备金 | 1/365法、1/24法、1/8法 |
| 已报案未决准备金 | 已报案但未赔付赔款对应准备金 | 链梯法、案均赔款法 |
| IBNR准备金 | 已发生未报案赔款准备金 | 链梯法、B-F法 |

### 3.2 未到期责任准备金（1/365法）

```
未到期责任准备金 = Σ(未到期保费 × 未到期天数 ÷ 365)
```

**示例**：保费3,650元，保期1年，已生效100天
→ 未到期准备金 = 3,650 × 265/365 = **2,652.05元**

### 3.3 链梯法（Chain Ladder Method）

**步骤**：
1. 编制赔款进展三角（事故年 × 开发年）
2. 计算各开发年的进展因子：`进展因子 = 第j+1年累计赔款之和 ÷ 第j年累计赔款之和`
3. 计算累计进展因子（从最新年到终极）
4. 预测最终赔款 = 最新累计赔款 × 累计进展因子
5. 未决准备金 = 最终赔款 - 最新累计赔款

**适用**：赔款发展规律稳定的成熟险种；对异常值敏感

### 3.4 B-F方法（Bornhuetter-Ferguson Method）

```
最终赔款 = 已报案赔款 + 期望最终赔款 × (1 - 累计进展比例)
期望最终赔款 = 已赚保费 × 期望赔付率
```

**适用**：新业务或数据量不足的险种（结合先验信息）

**示例**：
- 已赚保费100万；期望赔付率65%；已报案赔款20万；累计进展比例30%
- B-F最终赔款 = 20万 + 65万 × 70% = **65.5万元**
- IBNR准备金 = 65.5万 - 20万 = **45.5万元**

### 3.5 方法选择指引

| 方法 | 优先使用情形 | 注意事项 |
|------|------------|---------|
| 链梯法 | 赔款规律稳定、数据充足 | 对大赔案/异常值敏感，需先清洗 |
| 案均赔款法 | 索赔金额相对稳定、次数变化大 | 索赔次数预测误差会放大 |
| B-F法 | 新险种、数据不足前3年 | 期望赔付率需有可靠依据 |
| 多方法对比 | **⭐推荐：在精算报告中同时使用2-3种方法** | 对比差异，分析合理性 |

## 四、再保险定价实务

### 4.1 比例再保险

**成数分保（Quota Share）**：
```
分出保费 = 原保费 × 分出比例
摊回赔款 = 原赔款 × 分出比例
```

**溢额分保（Surplus Share）**：
```
分出比例 = (保险金额 - 自留额) ÷ (自留额 × 线数)
分出限额 = 自留额 × 线数
```

### 4.2 超赔再保险（Excess of Loss, XoL）

```
结构：起赔点（自留额）× of 责任限额（分保限额）
层：起赔点 at 责任限额
```

**期望赔款计算**：
- 方法1（历史法）：超历史赔款平均值 × 趋势调整因子
- 方法2（分布法）：E[max(0, 损失 - 起赔点)]

**帕累托分布超赔期望（形状参数α，起赔点d）**：
```
E[超赔赔款] = xm^α / (α-1) × d^(1-α)
```

### 4.3 停止损失再保险（Stop Loss）

```
再保险赔款 = max(0, 年度累计赔款 - 起赔点)
```

### 4.4 自留额度确定原则

1. 与公司资本实力匹配（通常不超过净资产的5-10%）
2. 参考历史损失经验（最大单一赔案历史值）
3. 考虑风险分散程度（同质风险池规模）
4. 比较再保险市场费率（自留 vs 分出的成本-效益）

## 五、损失分布与风险模型

### 5.1 常用损失分布

| 分布 | 参数 | 适用场景 | 特点 |
|------|------|---------|------|
| 正态分布 | μ, σ² | 大数定律下的平均损失 | 对称，可取负值（慎用） |
| 伽马分布 | α, β | 正偏态损失数据 | 正值，右偏，灵活 |
| 帕累托分布 | α, xm | 厚尾损失、巨灾风险 | 重尾，极端损失建模 |
| 对数正态分布 | μ, σ | 乘性效应的损失 | 对数对称，常用 |
| 威布尔分布 | λ, k | 生存时间分析 | 灵活形状 |

### 5.2 分布拟合步骤

```
Step 1: 数据探索（直方图、箱线图、QQ图）
Step 2: 参数估计（矩估计MOM / 最大似然MLE / 贝叶斯）
Step 3: 拟合优度检验（K-S检验、A-D检验、χ²检验）
Step 4: 模型选择（AIC/BIC + 对数似然值 + 残差分析）
```

### 5.3 现代机器学习方法（辅助精算）

| 方法 | 优势 | 适用场景 | 局限性 |
|------|------|---------|--------|
| 梯度提升（XGBoost/GBM） | 非线性、自动特征选择 | 分类费率厘定 | 可解释性弱 |
| 神经网络（ANN） | 复杂非线性、大规模数据 | 端到端纯保费预测 | 黑盒，监管认可度低 |
| 混合GLM+ANN | 兼顾可解释性与精度 | 平衡监管合规与精度 | 实现复杂 |

> ⚠️ **监管提示**：AI/ML定价模型须在精算报告中说明可解释性，监管对"黑盒"模型持审慎态度。当前主流仍为GLM，ML方法作为辅助验证。

## 六、数据管理与质量控制

### 6.1 精算数据来源三层级

| 层级 | 数据类型 | 说明 |
|------|---------|------|
| L1 公司内部数据 | 承保数据、理赔数据 | 最优先，但可能数据量不足 |
| L2 行业共享数据 | 行业协会统计、再保险公司共享 | 行业整体经验，来源可靠 |
| L3 外部公开数据 | 经济指标、自然灾害数据、统计年鉴 | 辅助判断趋势和环境因素 |

### 6.2 数据清洗要点

**常见问题处理**：
- **缺失值**：连续变量用中位数填充；分类变量用众数或单独设"未知"类
- **异常值**：Z-score法（|z|>3识别为异常）或IQR法（<Q1-1.5×IQR或>Q3+1.5×IQR）
- **重复值**：基于保单号+事故号去重
- **时间不一致**：统一时间戳格式，处理跨年度保单拆分

**精算底稿数据标准**（与开发检查清单 Draft-01~06 对应）：
- 每个步骤保留CSV格式原始数据底稿
- 字段规范：字段名（中英文）、数据类型、取值范围、备注
- 无数据时标注"基于精算师经验判断/类比法"

### 6.3 统计分析基础检验

| 检验类型 | 方法 | 用途 |
|---------|------|------|
| 均值差异检验 | t检验（两样本） | 比较两地区/险种的赔款差异显著性 |
| 分类变量独立性 | χ²检验 | 检验地区与行业是否独立 |
| 分布拟合 | K-S检验、A-D检验 | 验证损失数据是否符合假设分布 |
| 相关性分析 | Pearson/Spearman相关系数 | 筛选定价因子，避免多重共线性 |

## 七、产品开发流程中的精算职责

### 7.1 各阶段精算工作

| 开发阶段 | 精算职责 | 输出物 |
|---------|---------|--------|
| 步骤1（标的分析） | 初步可保性量化评估 | 风险单位定义、暴露量估计 |
| 步骤4（可研报告） | 损失数据分析、赔付率预测 | 第4-6章风险定量分析部分 |
| 步骤6（定价） | 基准费率厘定、因子体系 | 精算定价报告 + 费率表 |
| 步骤7（备案输出） | 精算责任人签字 + 数据底稿 | 精算报告正式版 + CSV底稿包 |

### 7.2 精算报告必要内容

1. **定价方法选择及依据**：说明为何选择纯保费法/损失率法/燃烧成本法，数据质量评价
2. **基础假设说明**：赔付率目标值、费用率、利润附加依据
3. **数据来源声明**：L1/L2/L3分级标注，无内部数据时说明类比依据
4. **风险因子体系**：因子选择依据（统计检验p值）、因子权重
5. **敏感性分析**：关键假设变动±10%/±20%时，保费和赔付率的变化范围
6. **精算结论**：建议基准费率、可接受费率区间、注意事项
7. **精算责任人声明**：签署人、签署日期、有效期

### 7.3 定价精算红线（监管合规底线）

| 监管要求 | 具体标准 |
|---------|---------|
| 预期赔付率底线 | 新产品首年预期赔付率 **≥ 40%** |
| 费率偏离控制 | 实际费率偏离行业平均水平不超过 **±30%** |
| 综合费用率上限 | 通常不超过 **40%**（金监局窗口指导） |
| 费率充足性 | 费率须能覆盖预期损失 + 合理费用 |
| 备案一致性 | 实际使用费率须与备案费率完全一致（"报行合一"） |

## 八、条款设计原则

### 8.1 五大核心保险原则

| 原则 | 含义 | 条款设计要点 |
|------|------|------------|
| 保险利益原则 | 被保险人对标的须有合法保险利益 | 第1章或第2章明确保险利益认定方式 |
| 损失补偿原则 | 赔偿以实际损失为限，不得额外获益 | 赔偿处理章节注明"比例赔付"或"重置价值" |
| 诚实信用原则 | 双方如实告知 | 第8章投保人义务章节 + 重要事项告知制度 |
| 射幸合同原则 | 保险事故须具有不确定性 | 责任免除章节排除确定性损失 |
| 风险定价原则 | 费率须反映风险水平 | 费率表设计须有差异化因子 |

### 8.2 条款设计核心规范

**保险责任设计**：
- 损失原因必须明确（列明制 vs 除外制）
- 区分直接损失/间接损失/费用损失
- 责任限额需设置每次事故限额 + 累计限额

**责任免除集中原则**：
- 所有除外责任必须集中在"责任免除"章节，不得分散
- 除外条款须显著标识（加粗/变色）
- 语言表述准确，无歧义

**释义规范**：
- 所有在条款中第一次出现的关键术语须在释义章节定义
- 释义需与行业通行定义一致
- 非通用术语须详细定义（尤其科技保险中的技术概念）

## 九、专业术语表

| 中文 | 英文 | 定义 |
|------|------|------|
| 风险单位 | Exposure Unit | 衡量风险暴露量的标准单位 |
| 索赔频率 | Claim Frequency | 单位风险单位的索赔次数 |
| 索赔强度 | Claim Severity | 平均每次索赔赔款金额 |

| IBNR | Incurred But Not Reported | 已发生未报案赔款准备金 |

| 溢额分保 | Surplus Share | 超过自留额部分分出的再保险 |
