# 非车险产品开发检查清单

> **适用范围**：非车险产品从概念设计到条款输出的全流程检查清单  
> **覆盖险种**：财产险、责任险、信用保证险、工程险、特殊风险险、科技保险、网络安全保险、绿色保险、低空经济保险等  
> **最后更新**：2026-03-31

---

## 步骤1：确定保险标的概念

### 1.1 可保五要素检查
- [ ] **合法性**：标的必须合法，非法财产不可保
- [ ] **确定性**：标的必须明确、可界定，具有明确边界
- [ ] **可估价性**：标的必须能用货币衡量价值
- [ ] **可转移性**：风险必须可以转移给保险人
- [ ] **偶然性**：损失发生必须具有不确定性

### 1.2 标的分类
```
保险标的
├── 有形财产（不动产、动产、在建工程）
├── 无形财产（预期利益、合同权益、知识产权、数据资产）
├── 法律责任（侵权责任、合同责任、法定责任）
└── 人身相关（雇主对雇员责任、医疗费用、人身伤亡赔偿）
```

### 1.3 标的估价方法
| 估价方式 | 适用场景 | 说明 |
|---------|---------|------|
| 重置价值 | 机器设备、建筑 | 出险时重新购置/建造成本 |
| 实际价值 | 二手资产、旧设备 | 重置价值 - 折旧 |
| 账面价值 | 存货、流动资产 | 按会计账面 |
| 协商价值 | 特殊标的 | 专业鉴定/协商确定 |
| 市场价值 | 标准化资产 | 参考市场交易价格 |

### 1.4 不可保标的识别与免责写入
> **关键规则**：步骤1识别的不可保标的，必须在步骤3直接写入责任免除条款。

- [ ] 土地、矿藏、水资源等自然资源 → 写入免责："土地、矿藏、水资源及其他自然资源"
- [ ] 文件、账册、图表等资料 → 写入免责："文件、账册、图表、技术资料等（可特约承保）"
- [ ] 货币、票证、有价证券 → 写入免责："货币、票证、有价证券（现金保险除外）"
- [ ] 违章建筑、非法占用财产 → 写入免责："违章建筑、非法占用的财产"
- [ ] 运输中物资 → 写入免责："运输途中的物资（货物运输保险承保）"
- [ ] 其他不可保标的 → 根据具体产品判断写入免责

### 1.5 标的数据查询（数据库接入）
- [ ] **行业标的数据**：尝试调用 `data_interfaces/industry_data_api.py` 获取行业标的价值参考/规模统计
- [ ] **灾害风险数据**：尝试调用 `data_interfaces/disaster_risk_api.py` 获取标的所在地区灾害风险系数
- [ ] **底稿保留**：查询结果保存为 CSV 格式（`data_drafts/step1_industry_data_YYYYMMDD.csv`）
- [ ] **无数据标注**：如无外部数据，记录"标的价值/规模数据基于精算师经验判断"

### 1.6 科技保险/新兴产业特殊标的处理
| 标的类型 | 估价方式 | 特殊注意事项 |
|---------|---------|-------------|
| 研发费用 | 实际支出/预算 | 需项目认定文件佐证 |
| 专利权 | 评估价值/诉讼费用 | 需确认专利有效状态 |
| 科研仪器 | 重置价值 | 需确认共享使用状态 |
| 技术成果 | 转化合同金额 | 需科技部门认定 |
| 碳汇资产 | 交易市场价格 | 需核证减排量证书 |
| 数据资产 | 评估价值/恢复成本 | 需数据资产登记/认证 |
| AI模型 | 开发成本/授权价值 | 需训练数据合规证明 |
| eVTOL飞行器 | 重置价值 | 需适航证和登记证 |

## 步骤2：明确投保人、被保险人身份

### 2.1 投保人资格审查
- [ ] 具有完全民事行为能力（自然人）/依法设立并存续（法人）
- [ ] 对保险标的具有保险利益
- [ ] 无不良信用记录、无保险欺诈历史
- [ ] 无重大未决诉讼、无欠缴保费记录

### 2.2 保险利益类型判定
| 利益类型 | 说明 | 典型场景 |
|---------|------|---------|
| 所有权利益 | 对标的享有所有权 | 财产所有人投保自有财产 |
| 使用权利益 | 对标的享有合法使用权 | 承租人投保租赁设备 |
| 保管权利益 | 对标的负有保管责任 | 仓储企业投保仓储货物 |
| 抵押权利益 | 对标的享有抵押权 | 银行投保抵押物 |
| 合同权益 | 基于合同产生的利益 | 总承包商投保工程 |
| 法定责任 | 法律规定需承担的责任 | 雇主责任、产品责任 |

### 2.3 被保险人范围确认
- [ ] 被保险人身份明确
- [ ] 投保人与被保险人关系清晰
- [ ] 被保险人同意投保（保险利益确认）

### 2.4 科技保险投保人特殊审查
- [ ] 科技型企业资质认定（高新技术企业/科技型中小企业）
- [ ] 项目认定文件（科技部门立项/备案/认定）
- [ ] 知识产权状态确认（专利有效性/登记状态）
- [ ] 资金来源合法性（财政资金需确认使用规定）

### 2.5 网络安全保险投保人审查
- [ ] 企业网络安全管理制度
- [ ] 数据安全等级保护合规证明
- [ ] 历史安全事件记录
- [ ] 安全防护措施（防火墙/加密/备份等）

### 2.6 绿色保险投保人审查
- [ ] 环保合规证明（排污许可证等）
- [ ] 绿色项目认证文件（绿色建筑/清洁能源/碳汇项目）
- [ ] 环境影响评价报告
- [ ] 历史环境违规记录

### 2.7 投保人信用数据查询（数据库接入）
- [ ] **信用评级数据**：尝试调用 `data_interfaces/credit_rating_api.py` 获取投保人信用评级/风险评估
- [ ] **底稿保留**：查询结果保存为 CSV 格式（`data_drafts/step2_credit_rating_YYYYMMDD.csv`）
- [ ] **无数据标注**：如无外部数据，记录"投保人信用状况基于核保人员人工审查"

## 步骤3：初步明确保险责任范围

### 3.1 产品类型选择
| 险种大类 | 适用场景 | 核心保障 |
|---------|---------|---------|
| 财产基本险 | 低风险工商企业 | 火灾、爆炸、雷击 |
| 财产综合险 | 一般工商企业 | 基本险 + 自然灾害 |
| 财产一切险 | 大型企业/外资企业 | 除责任免除外的一切风险 |
| 公众责任险 | 经营场所 | 第三方人身/财产损失 |
| 产品责任险 | 制造商/销售商 | 产品缺陷致人身/财产损害 |
| 雇主责任险 | 有雇员的企业 | 员工工伤赔偿 |
| 职业责任险 | 专业服务机构 | 专业过失赔偿责任 |
| 工程保险 | 建设工程 | 工程意外损失 + 第三者责任 |
| 信用保证险 | 贸易/贷款 | 违约/信用风险 |

### 3.2 责任设计模式

#### 3.2.1 传统财产险/责任险模式
- 财产险：列明风险式 或 一切险式（除外责任除外）
- 责任险：期内发生式 或 期内索赔式

#### 3.2.2 科技险责任设计模式
| 模式 | 适用产品 | 触发条件设计 |
|------|---------|-------------|
| 费用损失险 | 科技成果转化险、中试险 | 项目认定失败/未达预期成果而终止 |
| 知识产权维权险 | 专利执行险 | 专利被侵权，提起诉讼/仲裁/行政处理 |
| 设备损坏险 | 科研仪器共享险 | 意外损坏/灭失（机损险改造） |
| 技术责任险 | 自动控制系统责任险 | 系统故障致第三者人身/财产损失 |

#### 3.2.3 网络安全险责任设计
| 责任类型 | 保障内容 |
|---------|---------|
| 数据泄露 | 因安全事件导致的客户/个人信息泄露损失 |
| 网络勒索 | 勒索软件攻击导致的赎金、恢复费用 |
| 营业中断 | 因网络攻击导致的业务中断利润损失 |
| 数据恢复 | 数据恢复、系统重建费用 |
| 法律费用 | 监管调查、客户诉讼的法律辩护费用 |
| 危机公关 | 事件响应、通知成本、声誉修复费用 |

#### 3.2.4 绿色保险责任设计
| 子险种 | 核心责任 |
|--------|---------|
| 碳汇保险 | 碳汇资产损失/碳配额价格波动 |
| 绿色建筑保险 | 绿色技术/材料/设备损坏 |
| 清洁能源保险 | 风电/光伏/储能设施损坏 |
| 环境污染责任险 | 突发环境污染事故第三方赔偿 |
| 绿色信贷保证险 | 绿色项目贷款违约 |
| 生态修复保险 | 生态修复项目费用损失 |
| 气候指数保险 | 气象指标触发赔偿 |

#### 3.2.5 低空经济保险责任设计
| 子险种 | 核心责任 |
|--------|---------|
| 无人机机身险 | 机身意外损失/损坏（一切险） |
| 第三者责任险 | 无人机致第三方人身/财产损失 |
| eVTOL运营险 | 载人飞行器综合保障 |
| 无人机货运险 | 货物运输损失 |
| 低空基础设施险 | 起降场/充电桩/通信基站损失 |

### 3.3 除外责任设计矩阵

| 除外项 | 财产险 | 责任险 | 科技费用险 | 专利执行险 | 科研仪器险 | 网络安全险 | 绿色保险 | 低空经济险 |
|--------|--------|--------|-----------|-----------|-----------|-----------|---------|-----------|
| 故意行为/犯罪 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 战争/军事冲突 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 核辐射/核污染 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 自然磨损/渐进损坏 | ✅ | - | - | - | ✅ | - | ✅ | ✅ |
| 市场价格波动 | - | - | ✅ | - | - | - | 部分除外 | - |
| 行政/司法行为 | ✅ | - | ✅ | - | - | 监管罚款 | - | - |
| 投保前已知缺陷 | - | ✅ | ✅ | ✅ | ✅ | 已知漏洞 | ✅ | ✅ |
| 精神损害赔偿 | - | ✅ | - | - | - | - | - | - |
| 罚款/惩罚性赔偿 | - | ✅ | - | - | - | ✅ | - | - |
| 间接损失 | ✅ | 部分 | ✅ | - | - | ✅ | ✅ | - |

### 3.5 责任设计数据参考（数据库接入）
- [ ] **行业除外责任参考**：尝试调用 `data_interfaces/industry_data_api.py` 获取同业竞品除外责任条款数据
- [ ] **历史赔付数据**：尝试调用行业数据接口获取相关责任类型的历史赔付统计（用于除外责任范围校验）
- [ ] **底稿保留**：查询结果保存为 CSV 格式（`data_drafts/step3_exemption_data_YYYYMMDD.csv`）
- [ ] **无数据标注**：如无外部数据，记录"除外责任设计参照行业标准条款和监管要求"

### 3.6 赔偿限额设计
| 限额结构 | 适用场景 |
|---------|---------|
| 单一保额 | 财产险（保险金额） |
| 双层限额 | 责任险（每次事故限额 + 累计限额） |
| 三层限额 | 责任险（每次事故 + 累计 + 每人限额） |
| 六层限额 | 复杂责任险（每次事故 + 累计 + 财产损失 + 人身伤亡 + 每人伤亡 + 每人医疗） |
| 分项限额 | 专利执行险（调查费用限额 + 法律费用限额） |

## 步骤4：撰写可行性研究报告

### 4.1 八章节标准结构
1. **总体开发说明** — 监管依据三段式引用
2. **开发背景/开发目的** — 市场数据 + 政策依据
3. **法律依据** — 现行法律框架
4. **主要特点** — 投被保人 + 标的 + 责任免除 + 保险金额/期间
5. **风险分析及风险控制** — 核保/再保/操作风险管控
6. **经营模式** — 销售/承保/理赔/业绩监控
7. **（可选）保费测算/市场分析**
8. **总体结论**

### 4.2 监管依据标准三段式（必须引用）
```
国家金融监督管理总局2025年9月30日印发的《关于加强非车险业务监管有关事项的通知》（金发〔2025〕36号）
原银保监会2021年8月16日印发的《财产保险公司保险条款和保险费率管理办法》（银保监会令2021年第10号）
原保监会印发的《财产保险公司保险产品开发指引》（保监发〔2016〕115号）
```

### 4.3 操作风险管控模板（6个方面，可行性报告必含）
1. **产品开发阶段**：流程管控 + 人员培训
2. **产品备案环节**：标准化管理 + 复核机制
3. **产品披露环节**：合规管控
4. **产品系统配置**：流程管控 + 培训
5. **产品市场推广**：销售人员培训 + 销售行为管理
6. **产品承保**：规范核保流程 + 定期培训

### 4.4 可研报告数据来源指引

#### 4.4.1 外部数据接口（优先使用）
| 数据类型 | 接口脚本 | 数据底稿格式 |
|---------|---------|------------|
| 行业保费统计/赔付率 | `data_interfaces/industry_data_api.py` | CSV |
| 企业信用评级/风险评估 | `data_interfaces/credit_rating_api.py` | CSV |
| 地区灾害风险/风险系数 | `data_interfaces/disaster_risk_api.py` | CSV |

> **数据底稿保留要求**：所有外部数据查询结果必须保存为 CSV 格式文件，作为精算报告的附件留存。

#### 4.4.2 无外部数据库时的处理
> **如无外部数据库接入**：在可研报告和精算报告中标明数据来源：
> - "本报告费率测算基于精算师经验判断，参照同类产品市场水平"
> - "费率采用类比法确定，参照XXX公司同类产品费率"
> - "缺乏历史赔付数据，首年费率采用保守定价，预留调整空间"

#### 4.4.3 数据来源对照表
| 数据类型 | 公开数据源 | 外部API/系统 | 无数据时处理 |
|---------|-----------|------------|------------|
| 市场规模 | 统计局、行业协会年报 | 公司内部CRM/BI | 采用行业估算值 |
| 竞品分析 | 各保司官网、行业协会产品库 | 条款库检索系统 | 基于公开信息推断 |
| 历史赔付 | 无公开数据 | 公司核心系统精算数据 | 标注"精算师经验判断" |
| 行业风险 | 应急管理部灾害公报 | 灾害风险地图API | 参考行业平均水平 |
| 宏观经济 | 国家统计局、央行 | 宏观经济数据库 | 采用最新公开数据 |

## 步骤5：撰写保险产品条款

> ⚠️ **核心原则**：条款用语必须严格按照太平科技保险真实产品模板以及平安、太平洋保险的条款格式和用语习惯撰写。不得使用口语化、模糊化表述。

### 5.1 十三章节标准结构
1. **总则** — 合同构成、投保人/保险人定义
2. **保险标的** — 可保标的范围、特约可保标的、不可保标的
3. **保险责任** — 主险责任、附加险责任
4. **责任免除** — 原因除外、损失除外
5. **保险价值/保险金额/责任限额/免赔额（率）**
6. **保险期间** — 保险期限约定
7. **保险人义务** — 条款说明义务、赔偿义务
8. **投保人、被保险人义务** — 如实告知、安全维护、事故通知
9. **赔偿处理** — 赔偿方式、计算方法、代位求偿
10. **争议处理和法律适用** — 争议解决方式
11. **其他事项** — 合同变更、解除、终止
12. **释义** — 专业术语定义
13. **短期费率表** — 不足一年期间的费率

### 5.2 通用条款可直接引用
- 代位求偿权条款
- 重复保险分摊条款
- 保险事故通知条款
- 如实告知条款
- 权益转让条款
- 争议处理条款

### 5.3 条款特殊注意事项
- 费率表中的短期费率必须与条款中短期费率表一致
- 免赔额与免赔率的关系必须明确（加和 / 以高者为准）
- 保险价值的确定方式必须明确
- 赔偿计算公式必须完整清晰

### 5.4 条款用语规范（严格遵循太平科技/平安/太平洋真实条款格式）

> **适用范围**：所有主险条款撰写必须遵循以下用语规范。
> **依据来源**：太平科技保险6个真实产品条款、平安产险、太平洋产险条款。
> **参考文件**：`cases/` 目录下真实案例、`references/major_insurers_clauses_index.md` 五大公司条款索引。

#### 5.4.1 总体用语原则
- **正式法律文书风格**：条款表述应严谨、准确、无歧义
- **禁止口语化**：不得使用"大概"、"差不多"、"什么的"等口语表达
- **禁止模糊化**：不得使用"相关"、"适当的"、"合理的"等缺乏明确定义的修饰词，除非有明确标准可参照
- **统一人称**：使用"保险人"（指保险公司）、"投保人"、"被保险人"、"受益人"
- **统一时态**：保险责任使用"在保险期间内"、"发生"、"导致"等客观表述

**第七章 保险人义务（标准用语）**
```
第八条 订立保险合同时，采用保险人提供的格式条款的，保险人向投保人提供的投保单应当附格式条款，保险人应当向投保人说明保险合同的内容。对保险合同中免除保险人责任的条款，保险人在订立合同时应当在投保单、保险单或者其他保险凭证上作出足以引起投保人注意的提示，并对该条款的内容以书面或者口头形式向投保人作出明确说明。

第九条 本保险合同成立后，保险人应当及时向投保人签发保险单或者其他保险凭证。

第十条 保险事故发生后，保险人按照本保险合同的约定，认为有关的证明和资料不完整的，应当及时一次性通知投保人、被保险人补充提供。
```

**第八章 投保人、被保险人义务（标准用语）**
```
第十一条 订立保险合同，保险人就保险标的或者被保险人的有关情况提出询问的，投保人应当如实告知。

第十二条 投保人应按照本保险合同的约定交付保险费。

第十三条 被保险人应当遵守国家有关消防、安全、生产操作等方面的相关法律、法规及规定，加强管理，采取合理的预防措施，尽力避免或减少责任事故的发生，维护保险标的的安全。

第十四条 在保险期间内，保险标的危险程度显著增加的，被保险人应当及时通知保险人，保险人可以按照保险合同约定增加保险费或者解除保险合同。

第十五条 保险事故发生后，被保险人应该：
（一）尽力采取必要、合理的措施，防止或者减少损失，否则，对因此扩大的损失，保险人不负责赔偿；
（二）及时通知保险人，并书面说明事故发生的原因、经过和损失情况；
（三）保护事故现场，允许并且协助保险人进行事故调查；
（四）涉及诉讼、仲裁的，应提供相关法律文书。
```

**第九章 赔偿处理（标准用语）**
```
第十六条 保险事故发生时，被保险人对保险标的采取施救、保护等措施所支付的必要的、合理的费用，保险人在保险标的损失赔偿金额以外另行计算，最高不超过保险金额。

第十七条 保险标的发生全部损失，且保险金额等于或高于保险价值时，保险人按保险价值赔偿；保险金额低于保险价值时，保险人按保险金额赔偿。

保险标的发生部分损失，且保险金额等于或高于保险价值时，保险人按实际损失赔偿；保险金额低于保险价值时，保险人按保险金额与保险价值的比例承担赔偿责任。

第十八条 发生保险事故时，如有重复保险，保险人按照本保险合同相应保险金额与其他保险合同及本保险合同保险金额总和的比例承担赔偿责任。

第十九条 保险人收到被保险人的赔偿请求后，应当及时作出核定，并将核定结果通知被保险人；对属于保险责任的，在与被保险人达成赔偿协议后十日内，履行赔偿义务。保险合同对赔偿期限另有约定的，保险人应当按照约定履行赔偿义务。

第二十条 保险事故发生后，保险人未赔偿之前，被保险人放弃对第三者请求赔偿的权利的，保险人不承担赔偿义务。

保险人向被保险人赔偿保险金后，被保险人未经保险人同意放弃对第三者请求赔偿的权利的，该行为无效。

保险人向被保险人赔偿保险金后，在赔偿金额范围内代位行使被保险人对第三者请求赔偿的权利。
```


| 险种 | 主要因子 | 权重参考 |
|------|---------|---------|
| 费用损失险 | 项目成熟度(TRL等级)、行业领域、项目周期、团队资质、再保安排 | 成熟度30%+行业20%+周期20%+团队15%+再保15% |
| 专利执行险 | 专利类型、行业领域、维权历史、累计限额倍数 | 专利类型25%+行业25%+历史20%+限额30% |
| 科研仪器共享险 | 仪器价值、使用频率(年有效机时)、使用人类型、共享管理机制 | 价值30%+频率25%+使用人25%+管理20% |

### 6.6 网络安全险定价因子
| 因子类别 | 具体因子 | 说明 |
|---------|---------|------|
| 企业规模 | 营业额、员工数、用户数 | 资产规模越大风险越高 |
| 行业类型 | 金融/医疗/电商/制造业 | 高数据密集行业费率更高 |
| 安全投入 | 安全团队规模、安全工具 | 安全投入越高费率越低 |
| 历史事件 | 过去3年安全事件数量 | 有事件记录费率上升 |
| 数据敏感度 | 个人信息/金融数据/医疗数据 | 数据敏感度越高费率越高 |

### 6.7 绿色保险定价因子
| 因子类别 | 具体因子 | 说明 |
|---------|---------|------|
| 项目认证等级 | 绿色建筑星级/清洁能源类型 | 认证等级越高风险可控 |
| 项目规模 | 装机容量/碳汇面积/建筑面积 | 规模影响风险暴露 |
| 技术成熟度 | TRL等级/设备品牌 | 成熟技术费率更低 |
| 地理环境 | 自然灾害风险区域 | 灾害高发区费率上升 |
| 运营管理 | 运维团队资质/管理体系 | 专业运维降低风险 |

### 6.8 再保险对定价的影响
| 分保方式 | 成本影响 | 适用场景 |
|---------|---------|---------|
| 成数分保 | 分出保费 × 分出比例 × 附加费率 | 统一处理大量业务 |
| 溢额分保 | 自留额以上部分 × 临分费率 | 大额风险分散 |
| 临时分保 | 单笔谈判，费率波动大 | 特殊/高额风险 |
| 合约再保 | 长期稳定，费率较优 | 常规业务安排 |

### 6.9 信用保证险定价逻辑
```
预期损失率 = PD（违约概率）× LGD（违约损失率）× EAD（违约风险暴露）
费率 = 预期损失率 / (1 - 费用率 - 利润率)
```

### 6.10 费率合理性校验
- [ ] 费率偏离行业平均水平 ≤ ±30%（金发〔2025〕36号要求）
- [ ] 首年预期赔付率在 50%-70% 区间
- [ ] 费率与风险因子权重计算结果一致
- [ ] 短期费率表与条款附录一致
- [ ] 费率表层次完整（基准费率 + 调整系数）

### 6.11 定价数据管理与底稿保留
- [ ] **数据库接入检查**：尝试调用 `data_interfaces/` 下的行业数据接口获取费率参考
- [ ] **底稿保留**：所有外部数据查询结果保存为 CSV 格式（含查询时间、数据来源、原始值）
- [ ] **无外部数据标注**：如无法获取外部数据，在精算报告中明确标注"基于精算师经验判断/类比法"
- [ ] **数据可追溯**：费率表中的每个数值应能追溯到数据底稿中的来源

## 步骤8：附加险开发流程（无需可研报告）

> **前提条件**：必须已有对应主险条款，附加险不能独立存在。

### 8.1 确认主险条款
- [ ] 明确附加险所依附的主险名称和条款编号
- [ ] 确认主险的保险标的范围
- [ ] 确认主险的保险责任和责任免除范围
- [ ] 确认主险的赔偿限额结构和免赔设定
- [ ] 核对主险条款是否已声明"可附加本附加险条款"

### 8.2 确定附加险保障范围
- [ ] 明确附加险类型（风险扩展型/费用扩展型/限额扩展型/场景扩展型）
- [ ] 确定触发条件（独立触发/需与主险事故同时发生）
- [ ] 设定附加险独立的赔偿限额（不突破主险限额）
- [ ] 设定附加险独立的免赔额（率）
- [ ] **不可保标的处理**：如附加险涉及不可保标的，直接写入除外责任

### 8.3 除外责任设计
- [ ] 继承主险全部除外责任（附加险除外责任 ≥ 主险除外责任）
- [ ] 增加附加险特有的除外项
- [ ] 除外责任措辞与主险保持一致

### 8.4 附加险定价
- [ ] 附加险基准费率确定（参照主险费率 × 附加系数，或独立定价）
- [ ] 附加系数参考：风险扩展型(0.2-0.5)、费用扩展型(0.05-0.15)、限额扩展型(0.1-0.3)
- [ ] 附加险调整因子设定（通常比主险简化，3-5个因子）
- [ ] 附加险费率表生成
- [ ] **定价数据底稿**：保留为 CSV 格式
- [ ] **无外部数据标注**：如无数据，标注"参照主险费率比例确定"

### 8.5 附加险条款撰写（精简版，6-8章节）
1. **总则** — 引用主险条款，声明附加险依附关系
2. **保险责任** — 附加险扩展的保障内容
3. **责任免除** — 继承主险 + 附加险特有除外
4. **赔偿限额与免赔额** — 附加险独立的限额和免赔
5. **赔偿处理** — 引用主险赔偿处理规则
6. **其他事项** — 合同变更、终止规则
7. **释义** — 附加险特有术语定义

### 8.6 关联关系确认
- [ ] 主险条款中增加"本保险可附加XXX附加险"声明
- [ ] 附加险条款中明确"本附加险须附加于XXX主险"
- [ ] 费率表中主险费率与附加险费率的匹配关系
- [ ] 短期费率表一致性检查
- [ ] 主险终止时附加险自动终止规则确认

### 8.7 输出附加险条款和费率表
- [ ] 附加险条款Word文档（精简格式化条款）
- [ ] 附加险费率表（可与主险费率表合并，也可独立输出）
- [ ] 附加险定价数据底稿CSV文件
