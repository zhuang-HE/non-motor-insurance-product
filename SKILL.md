# 非车险产品开发技能

覆盖非车险产品从概念设计到条款输出的全流程开发工具。

> **详细参考**（完整条款标准用语、可行性报告模板、242产品路由表、Word 格式代码）→ `references/clause-standards-and-templates.md`

## ⚡ 执行规则（强制遵守）

### 脚本调用规则
1. **禁止重新编写脚本**：必须使用 `scripts/` 目录下现有脚本，不得重新生成 Python 代码
2. **顺序执行**：必须按 7 步顺序执行，不可跳过或颠倒
3. **参数传递**：步骤间通过 JSON 文件传递数据，不得依赖对话记忆
4. **统一入口**：优先使用 `scripts/workflow_controller.py` 执行完整流程
5. **状态持久化**：使用状态文件保持执行进度，支持中断恢复

### 格式遵守规则
1. **Word 格式**：必须严格遵循 `templates/word_format.json` 格式配置
2. **条款用语**：必须使用 `references/clause-standards-and-templates.md` 中的标准表述模板
3. **报告结构**：可行性报告必须包含 8 章节结构和三段式监管引用
4. **输出验证**：每步完成后必须通过 `scripts/validators.py` 验证输出格式

### 验证要求
- 每步完成后必须验证输出格式
- 最终 Word 文档必须通过格式检查
- 使用 `scripts/workflow_controller.py` 时可自动验证

## 📋 七步核心流程

```
阶段一：市场调研 → 阶段二：条款撰写 → 阶段三：费率厘定 → 阶段四：备案上市
   ↓              ↓              ↓              ↓
 可研报告        条款文件        费率表         备案材料
 (v1.0)         (v1.0)         (v1.0)        完成备案
```

| 步骤 | 名称 | 输出物 | 核心内容 |
|------|------|--------|---------|
| **1** | 确定保险标的概念 | — | 合法性审查 + 五性可保性分析 + 标的分类估价 |
| **2** | 明确投保人/被保险人 | — | 资格审查 + 保险利益认定（7 种权益类型） |
| **3** | 明确保险责任范围 | — | 基本险/综合险/一切险选型 + 主险+附加险 + 除外体系 |
| **4** | 撰写可研报告 | `可研报告.md` | 8 章节结构，含监管三段式引用 + 操作风控 6 方面 |
| **5** | 撰写完整条款 | `XX保险条款.md` | 13 章节标准结构（总则→释义） |
| **6** | 产品定价和风险因子 | `XX费率表.md` | 定价方法选择 + 因子权重 + 费率表制作 |
| **7** | 输出 Word 文档 | `.docx` | 条款文档 + 费率表的 Word 格式输出 |

## 🔒 零、监管合规框架

### 核心监管法规

| 层级 | 法规名称 | 适用 |
|------|---------|------|
| 部门规章 | 《财产保险公司保险条款和保险费率管理办法》（金发〔2025〕36号） | 条款费率制定与备案 |
| 部门规章 | 《财产保险公司保险产品开发指引》（银保监发〔2021〕10号） | 产品开发合规性 |
| 规范性文件 | 《关于加强非车险业务监管有关事项的通知》 | 非车险业务经营 |

### 监管红线（5 条禁止）

1. 不得开发违反法律法规或损害社会公共利益的产品
2. 不得开发条款模糊、存在歧义的产品
3. 不得开发费率低于合理精算基础的产品
4. 不得使用未经备案的条款费率
5. 不得在条款中设置未显著提示的加重被保险人责任的免责条款

### 备案材料

| 材料 | 格式 | 说明 |
|------|------|------|
| 保险条款 | .docx + PDF | 13章节完整结构 |
| 费率表 | .xlsx + PDF | 含精算测算依据 |
| 可研报告 | .docx | 8章节标准结构 |
| 精算报告 | .docx | 定价方法与假设 |

## 一、条款标准化体系

### 13 章节标准结构

| # | 章节 | 编号 | 核心 |
|---|------|------|------|
| 1 | 总则 | 第1-2条 | 订立目的、合同组成、定义解释 |
| 2 | 保险标的 | 第3-4条 | 标的范围、除外标的 |
| 3 | 保险责任 | 第5-6条 | 主险责任、附加责任、赔偿计算 |
| 4 | 责任免除 | 第7-8条 | 通用除外、专项除外 |
| 5 | 保险价值/金额/免赔额 | 第9-10条 | 金额确定、免赔约定 |
| 6 | 保险期间 | 第11-12条 | 期间约定、续保条款 |
| 7 | 保险人义务 | 第13-14条 | 核保、赔偿、信息保密 |
| 8 | 投保人/被保险人义务 | 第15-17条 | 如实告知、安全维护、出险通知 |
| 9 | 赔偿处理 | 第18-20条 | 索赔材料、核赔时限、支付时限 |
| 10 | 争议处理和法律适用 | 第21-22条 | 协商/仲裁/诉讼、法律适用 |
| 11 | 其他事项 | 第23条 | 保全变更、合同解除 |
| 12 | 释义 | 第24条 | 术语定义 |
| 13 | 短期费率表 | 附表 | 日费率、短期费率 |

### 险种特有板块速查

**企财险：** 物质损失 + 机器损坏(可选) + 营业中断(可选) + 重置/实际价值

**责任险：** 第三者责任 + 雇主(可选) + 公众(可选) + 产品(可选) + 法律费用

**特殊风险（无人机/机器人）：** 机身险 + 三者责任 + 适航条件 + 飞行区域限制

## 二、可行性研究报告结构

### 8 章节骨架

```
一、总体开发说明（监管依据三段式 + 开发目的）
二、开发背景（市场规模 + 政策支持 + 法律依据）
三、产品主要特点（投保人/责任/免责/金额/期间）
四、保险费率及保费测算（基准费率 + 调整系数 + 预期）
五、风险分析及风控（逆选择 + 道德 + 操作风控6方面）
六、经营模式（销售/承保/理赔/监控）
七、（可选）市场类似产品分析
八、总体结论
```

**必须包含：**
- 监管依据三段式引用（金发〔2025〕36号 + 银保监发〔2021〕10号）
- 操作风险管控 6 方面（开发→备案→披露→系统→推广→承保）

## 三、精算定价

### 定价方法选择

| 方法 | 适用场景 | 要求 |
|------|---------|------|
| 纯保费法 | 有充足历史数据 | 损失分布拟合 + 风险边际 |
| 损失率法 | 有参考费率 | 基于参考费率的损失率调整 |
| Burning Cost法 | 经验数据有限 | 参考历史赔付 + 风险调整 |
| 类比法 | 新产品开发 | 找最相似产品 + 专家判断 |

### 风险因子权重

**财产险：** 建筑 25% + 消防 20% + 占用性质 25% + 年代 15% + 安全管理 15%

**责任险：** 行业 35% + 区域 20% + 历史 20% + 规模 15% + 限额免赔 10%

### 典型基准费率（来自 242 产品库）

| 产品 | 基准费率 | 计费基础 |
|------|---------|---------|
| 专利执行险-调查费 | 4.9% | 累计赔偿限额 |
| 专利执行险-法律费 | 5.0% | 累计赔偿限额 |
| 无人机机身险 | 16.0% | 机身价值 |
| 无人机第三者责任 | 1.0% | 赔偿限额 |
| 财产基本险 | 0.8‰-2.5‰ | 保险金额 |
| 财产一切险 | 1.5‰-5.0‰ | 保险金额 |
| 安全生产责任险 | 1.5‰-6.0‰ | 从业人员数 |
| 雇主责任险 | 1.0‰-8.0‰ | 工资总额 |
| 建筑工程一切险 | 1.5‰-4.5‰ | 工程造价 |

## 四、质量控制

### 条款合规检查清单
- [ ] 符合《财产保险公司保险条款和保险费率管理办法》
- [ ] 符合《财产保险公司保险产品开发指引》
- [ ] 条款无歧义、表述清晰；责任免除明确无争议
- [ ] 免责条款已显著提示（加粗/颜色标注）
- [ ] 符合《保险法》如实告知、不可抗辩等规定

### 费率合规检查清单
- [ ] 费率厘定有精算依据；符合"报行合一"
- [ ] 费率表格式规范、计算准确；最低保费合理

### 审核流程
```
撰写 → 内部审核 → 法务合规审核 → 精算审核 → 分管领导审批 → 备案
```

## 自我修复与迭代验证

> 借鉴 Insurance-Skills 精算建模的自我修复机制，增加质量控制迭代。

### 迭代验证流程

```
┌─────────────────────────────────────────────────────────────┐
│  Step X 执行 → 质量检查 → 通过？                            │
│       ↓ 否                                                 │
│  问题分类 → 修复方案 → 回退 Step？→ 重新执行               │
│       ↓                                                     │
│  迭代计数 +1 → 迭代 > 2？→ 标记问题，输出待解决清单        │
└─────────────────────────────────────────────────────────────┘
```

### 回退触发条件

| 严重性 | 问题类型 | 处理方式 | 迭代上限 |
|--------|---------|---------|---------|
| 🔴 阻断级 | 责任与免责矛盾 | 回退 Step 5 | 2次 |
| 🔴 阻断级 | 费率低于精算基础 | 回退 Step 6 | 2次 |
| 🟡 警告级 | 章节缺失/格式错误 | 回退该步骤 | 3次 |
| 🟢 建议级 | 用语不规范 | 标注但不回退 | 无限制 |

### 修复验证清单

每次回退后必须重新验证：
- [ ] 该问题已修复
- [ ] 修复未引入新问题
- [ ] 上下游步骤不受影响

## 七、常见陷阱与反模式

> 借鉴 Insurance-Skills 质量体系，整理保险产品开发高频错误。

### 条款撰写常见陷阱

| # | 反模式 | 严重性 | 快速修复 |
|---|--------|--------|----------|
| 1 | 责任范围与除外责任重叠 | 🔴 高 | 使用条款用语矩阵的"责任边界检测"，逐条核对 |
| 2 | 赔偿限额表述歧义 | 🔴 高 | 明确区分"每次事故限额"vs"累计限额"vs"每人限额" |
| 3 | 费率低于精算基础 | 🔴 高 | 必须经过费率测算，不可凭经验定价 |
| 4 | 缺少监管依据引用 | 🟡 中 | 每条责任须附法规依据，可研报告必须三段式引用 |
| 5 | 免责条款未显著提示 | 🟡 中 | 加粗或颜色标注，确保消费者能识别 |

### ❌ 错误示例

**责任范围歧义：**
```markdown
# 错误写法
保险人对因火灾造成的损失负责赔偿，但以下情况除外：
（一）因地震造成的损失；
（二）保险标的因火灾造成的间接损失。

# 问题：地震免责与火灾责任矛盾，逻辑不清
```

**限额表述不清：**
```markdown
# 错误写法
保险人对每次事故的赔偿金额不超过100万元。

# 问题：未明确"每次事故"还是"累计"，未区分人身/财产限额
```

### ✅ 正确写法

```markdown
# 正确写法
第六条 赔偿限额
（一）累计赔偿限额：人民币[X]万元
（二）每次事故赔偿限额：人民币[X]万元
（三）每次事故每人赔偿限额：人民币[X]万元
（四）每次事故财产损失赔偿限额：人民币[X]万元

第七条 保险责任
在保险期间内，对因下列原因造成保险标的的损失，负责赔偿：
（一）火灾、爆炸；
（二）雷击、暴雨、洪水、台风...
```

## ⚙️ 核心规则

1. **合规先行** — 所有产品必须符合监管框架，引用三段式监管依据
2. **标准用语** — 条款表述使用标准化语言，确保严谨无歧义
3. **13 章结构** — 条款必须遵循 13 章节标准结构，不可省略核心章节
4. **精算有据** — 费率厘定必须有合理精算基础，不得低于成本
5. **免责提示** — 责任免除条款必须显著提示（字体加粗或颜色标注）
